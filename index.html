How It Works

The pipeline

Each startup receives a personalised, pre-filled form via a tracked link. Every click and submission is logged; every submission triggers an automated thank-you.

Capabilities

System features

Built to the same design language as other KLE-CTIE admin portals — consistent branding, custom dialogs, and editable configuration without touching code.

✉

Editable Sender Config

Sender name, reply-to, subject, CTA label and deadline are all editable from a single Config row — no code changes needed to re-run the campaign.

🔗

Per-Startup Tracking

Each invitation carries a unique token. Clicks are logged with timestamp to a ClickLog sheet, powering an accurate invite → click → submit funnel.

✓

Field Validation

Email, 10-digit phone, and URL fields are validated at entry. Names are auto–sentence-cased and emails lower-cased on submission for a clean dataset.

📊

Live Dashboard

Password-gated dashboard on GitHub Pages pulls live data via the Apps Script endpoint — KPIs, conversion funnel, domain & stage charts, and CSV export.

Challenges Solved

Sandbox redirect blocking

Browser policy blocks auto-navigation from the Apps Script iframe. Resolved with a meta-refresh + user-gesture fallback link that works across all browsers.

File-upload API limits

Native file-upload questions require Google sign-in, excluding non-Gmail founders. Replaced with a validated logo-URL field for universal access.

Effort Summary

From file to system

The build progressed from a raw spreadsheet to a fully automated, tracked, and reviewable system — with each technical obstacle diagnosed and resolved along the way.

Stage 1

Data cleaning & schema design

Raw startup list trimmed, sentence-cased, validated and structured into a Master + Config + Lists + ClickLog workbook.

Stage 2

Form generation & validation

Programmatic 10-field form with email, phone and URL validation, dropdowns for domain and stage, and pre-fill per startup.

Stage 3

Tracking, mailers & automation

Tracked redirect, branded invitation and thank-you emails, editable sender config, and onFormSubmit normalisation.

Stage 4

Analytics dashboard

Password-gated dashboard on GitHub Pages with live funnel, charts, searchable status table and CSV export.

In Summary

A reusable, zero-cost foundation

One spreadsheet, one script, one dashboard — re-runnable every year by changing a single Config row. Built to last beyond any one campaign, and to be handed over cleanly.
